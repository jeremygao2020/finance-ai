
  # AI记账看板设计

  #为什么要做？
1. 当前AI推荐银行流水、发票凭证模板不准确，记账规则未持续迭代，目前准确率不足90%，
2. 推荐不准确，用户需要人工调整，用户反馈给金蝶服务人员，传递给研发也未作具体的专项任务跟进导致解决时间周期长，且存在漏处理的情况
设计目标
1. 针对用户修订过的凭证规则，可快速的进行检查并标注，并将标注的结果反哺给大模型，便于下次能精准识别
2. 可直观的看到每一家企业银行流水、发票推荐的准确率
使用角色
1. 财税产品经理：快速标注，沉淀规则，提升AI记账准确率
2. 产品管理者：监控AI推荐的准确率，基于准确率目标推动产品进行改进
产品设计
AI记账看板：
1. 检索条件：代账公司名称（支持模糊搜索）、代账公司eID、企业名称（支持模糊搜索）、服务ID
2. 列展示：代账公司名称、公司名称、AI记账账套数、AI推荐准确率-流水、AI+规则推荐准确率-流水、AI推荐准确率-发票、AI+规则推荐准确率-发票、近1月AI推荐流水记账规则数量、近1月AI推荐发票记账规则数量、最后一次AI推荐实践

发票标注：
1. 检索条件：代账公司名称（支持模糊搜索）、代账公司eID、企业名称（支持模糊搜索）、服务ID、发票项目名称、发票备注、发票起始日期、发票结束日期
2. 批量处理：支持批量审核规则、修改规则
3. 列展示：代账公司名称（支持模糊搜索）、代账公司eID、企业名称（支持模糊搜索）、服务ID、AI推荐的凭证规则（显示凭证借贷方分录，若分录超过2行则通过tips显示完成的凭证分录）、用户修正后的凭证规则（显示凭证借贷方分录，若分录超过2行则通过tips显示完成的凭证分录）、状态（默认为未审核、已审核）、当前处理人、审核人、审核时间、发票明细（显示商品名称，若超过2行则通过tips显示）、发票备注、发票日期、操作（确认规则、调整规则）
4. 列展示-操作，列操作包含（审核规则、修改规则），点击审核规则弹出审核规则的提示同时更新状态为已审核，点击修改规则，则弹出规则修改框，规则包含行业、业务类型、关键词、借贷方凭证模板
流水标注：
1. 检索条件：代账公司名称（支持模糊搜索）、代账公司eID、企业名称（支持模糊搜索）、服务ID、流水摘要、流水备注、流水对方名称、流水起始日期、流水结束日期
2. 批量处理：支持批量审核规则、修改规则
3. 列展示：代账公司名称（支持模糊搜索）、代账公司eID、企业名称（支持模糊搜索）、服务ID、AI推荐的凭证规则（显示凭证借贷方分录，若分录超过2行则通过tips显示完成的凭证分录）、用户修正后的凭证规则（显示凭证借贷方分录，若分录超过2行则通过tips显示完成的凭证分录）、状态（默认为未审核、已审核）、当前处理人、审核人、审核时间、流水摘要、流水备注、流水对方名称、流水日期、操作（确认规则、调整规则）
4. 列展示-操作，列操作包含（审核规则、修改规则），点击审核规则弹出审核规则的提示同时更新状态为已审核，点击修改规则，则弹出规则修改框，规则包含行业、业务类型、关键词、借贷方凭证模板

规则管理
1. 可分别添加流水、发票记账规则，添加规则包含行业、业务类型、借贷方凭证模板，每一条规则支持维护关键词
2. 默认展示商贸行业的记账凭证规则（凭证模板）
3. 规则支持检索，支持列表查看

  ## Running the code

  Run `npm i` to install the dependencies.

  Run `npm run dev` to start the development server.
